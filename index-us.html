Services Management Tool

GUI for Remote Service Manipulation

About the Project

This project consists of a graphical user interface (GUI) developed in Python, using the Ttkbootstrap library, for remote manipulation of services in Production and Staging environments of a Senior HCM SaaS infrastructure.

The tool allows querying, starting, and stopping services associated with a specific client, simplifying service management across multiple servers.

Key Features

Remote Service Query

The application performs remote queries to check the status of services using PowerShell commands via the subprocess function, presenting the results in an interactive table.

Differentiated Environments

The user can select whether to perform the operation in the Production or Staging environment, ensuring greater control over the environments.

Service Filtering

The interface has a filtering system where the real service names are masked with descriptive names. Ex: "seniorinst" is presented as "Senior Installation Information Service".

Service Manipulation

The user can select specific services or choose to manipulate all services at once. From this selection, they can start or stop services directly from the interface.

Technologies Used

Python 3

The main language used in the project for application development.

Ttkbootstrap

A native Python library for creating modern graphical interfaces.

Subprocess

Module used to execute PowerShell commands remotely.

PowerShell

Scripting language for performing queries and operations on remote servers.

User Interface

Control Buttons

Query, start, or stop services via intuitive buttons

Dynamic Table

Presents results with service name (masked) and current state

Interactive Combobox

Select a specific service or all services for an operation

Script Logic

Query Services

Using the "Query Services" button, the application collects active services on the remote server, applying filters for the selected client and environment. The PowerShell command is executed remotely, and the results are displayed in the table.

Start/Stop Services

The user can start or stop services individually or in bulk. To do this, they choose the desired service from the combobox and click "Start Service(s)" or "Stop Service(s)", triggering the PowerShell command that performs the action on the remote server.

Use Case Example

3

PowerShell Script for Process Management

Sep 13, 2024 Â· Technical Article

Introduction

In the context of HCM systems administration, managing processes remotely is a critical task to ensure the smooth operation of client environments. One of the challenges is monitoring and terminating running processes efficiently, without negatively impacting service operations.

This PowerShell script was developed to perform remote verification and termination of processes, allowing operator interaction and ensuring that only specific client processes are affected.

Script Features

Operator Interaction

The script prompts the user for important information, such as the remote server name, the client name, and the environment type (production or staging), making it flexible for different scenarios.

Remote Query

Using the Invoke-Command, the script queries the remote server to identify running processes related to the client. The applied logic filters processes by the path where they are running, ensuring accurate identification.

Safe Termination

After listing the running processes, the operator can choose to terminate one or more processes. This is done in a controlled manner, where the process ID is provided by the user and the termination is performed remotely using the Stop-Process command.

Flow Control

The script runs continuously, allowing the operator to perform multiple checks and terminations without having to restart it manually. After each cycle, the operator can choose whether to restart the script or exit.

Execution Flow

1

The user enters the remote server name and the client name.

2

The script queries if there are any running processes for the client on the remote server.

3

If processes are found, they are listed for the operator.

4

The operator can choose to terminate processes individually by providing the process ID.

5

The script asks if the operator wants to continue or terminate more processes.

Benefits

Automation and Efficiency

Reduces the need for manual interventions and automates the search for client-related processes

Flexibility

Can be quickly adapted for different environments (production or staging) and different clients

User-Friendly Interaction

The operator interface is simple and clear, making it easy for system administrators to use

Full Code

# This is the original PowerShell code, left untranslated as it is a code block.
While ($true) {
    Clear-Host;
    Write-Host "Mega Cloud HCM - Checks running processes by client..." -ForegroundColor Green

    Write-Host "Enter the remote server name:" -ForegroundColor Cyan
    $computerName = Read-Host

    # CLIENT DEFINITION
    Write-Host "What is the client's name?" -ForegroundColor Cyan
    $nomeCli = Read-Host

    Write-Host "Do you know the client's HCM code? (Y)Yes, (N)No" -ForegroundColor Cyan
    $codCliQ = Read-Host;

    if ($codCliQ -eq "N") {
        # If the client code is not known
        Invoke-Command -ComputerName $computerName -ScriptBlock {
            param($nomeCli)
            Get-WMIObject win32_service | Where-Object { $_.pathname -like "*$nomeCli*" -and $_.name -like "*SeniorInstInfoService*" } | Format-Table -AutoSize PSComputerName, Name, State, PathName
        } -ArgumentList $nomeCli

        Write-Host "Enter the client code:" -ForegroundColor Cyan
        $codCli = Read-Host
    } else {
        Write-Host "Confirm the code..." -ForegroundColor Cyan
        $codCli = Read-Host
    }

    Write-Host "What is the environment type? (p)Production, (h)Staging" -ForegroundColor Cyan
    $tipAmb = Read-Host
    $cliente = "$nomeCli" + "_" + "$codCli" + "_" + "$tipAmb"

    Write-Host "Querying processes..." -ForegroundColor Cyan
    
$nenhumProcesso = $false

# QUERY RUNNING PROCESSES
Invoke-Command -ComputerName $computerName -ScriptBlock {
    param($cliente)

    # Store processes in a variable
    $processos = Get-Process -Name * | Where-Object { $_.path -like "*$cliente*" }

    # Check if the process list is empty
    if ($processos) {
        # If there are processes, display them in a table format
        $processos | Format-Table -AutoSize ID, ProcessName, Path
    } else {
        # Otherwise, display the message and set a flag to skip termination
        Write-Host "No processes found for client $cliente" -ForegroundColor Yellow
        return $true
    }

} -ArgumentList $cliente -OutVariable resultado

    $nenhumProcesso = $resultado

# Check if no process was found
if ($nenhumProcesso -eq $true) {
    # Skip termination and go directly to the repeat loop
    Write-Host "Do you want to restart the script? Yes(Y) No(N)" -ForegroundColor Cyan
    $restartScript = Read-Host
    if ($restartScript -eq "Y") {
        continue
    } else {
        break
    }
}

# If processes were found, continue the flow to terminate processes
Write-Host "Do you want to terminate any process? (Y)Yes, (N)No" -ForegroundColor Cyan
$processo = Read-Host

if ($processo -eq "Y") {
    do {
        Write-Host "Enter the ID of the process you want to terminate" -ForegroundColor Cyan
        $comando = Read-Host

        # TERMINATE THE DESIRED PROCESS
        Invoke-Command -ComputerName $computerName -ScriptBlock {
            param($comando)
            Get-Process | Where-Object { $_.id -eq $comando } | Stop-Process -Force
        } -ArgumentList $comando

        Write-Host "Process terminated successfully!!" -ForegroundColor Cyan

        # Display remaining processes
        Invoke-Command -ComputerName $computerName -ScriptBlock {
            param($cliente)
            Get-Process -Name * | Where-Object { $_.path -like "*$cliente*" } | Format-Table -AutoSize ID, ProcessName, Path
        } -ArgumentList $cliente

        # Ask if you want to terminate more processes
        Write-Host "Do you want to terminate more processes? (Y)Yes, (N)No" -ForegroundColor Cyan
        $maisProcessos = Read-Host

    } while ($maisProcessos -eq "Y")
}

# REPEAT LOOP
Write-Host "Do you want to restart the script? Yes(Y) No(N)" -ForegroundColor Cyan
$restartScript = Read-Host
if ($restartScript -eq "Y") {
    continue
} else {
    break
}

}

Exit

Conclusion

PowerShell automation is an excellent tool for ensuring control and monitoring of processes in corporate environments, providing more security and efficiency to the IT team.

This script offers a powerful and automated solution for managing remote processes in HCM environments, contributing to the stability and efficiency of the services provided.

Share Correction Script

Sep 11, 2024 Â· Technical Article

Project Goal

In my recent project, I developed a robust PowerShell script to manage folder shares across multiple production servers, aiming to optimize administration and ensure that important folders were properly shared.

To ensure that all production folders on specific servers are adequately shared and that non-existent shares are created as needed.

What the script does

Query and Filtering

The script connects to remote servers and gets a list of folders in the D:\ path that meet specific criteria.

Servers: OCSENAPLH01, OCSENAPL01, OCSENAPL02, OCSENAPL03, OCSENAPL04

Criteria: Names containing "_p" and excluding "OLD" and "Test"

Share Verification

For each folder, it checks if a corresponding share already exists. If there is no existing share, the script creates a new one.

Permission Configuration

The script collects the current permissions of each folder and classifies user groups according to defined rules.

HR HCM Compliance Sapiens

Maintains existing NTFS permissions while creating new shares

Interface and Repetition

After running on all servers, the script offers the option to restart execution, allowing for continuous administration as needed.

Example Output

Running query on server OCSENAPL01

The folder Project_X is shared.

The folder TestFolder is not shared.

The folder TestFolder was shared as \\OCSENAPL01\TestFolder$.

Benefits

Automation

Reduction of manual work and human errors by ensuring all necessary folders are shared correctly

Efficiency

The ability to manage multiple servers simultaneously and consistently

Flexibility

Ease of adjusting filtering criteria and permission groups according to future needs

Main Code Structure

# This is the original PowerShell code, left untranslated as it is a code block.
# Main loop for all servers
while ($true) {
    Clear-Host
    Write-Host "Script to query and correct HCM PRODUCTION shares" -ForegroundColor Green

    # For each server (OCSENAPLH01, OCSENAPL01-04)
    $nomeServidor = "SERVER_NAME"
    
    # Get folders that meet the criteria
    $listaPastas = Invoke-Command -ComputerName $nomeServidor -ScriptBlock {
        Get-ChildItem -Path D:\* -Directory | Where-Object {
            $_.Name -like "*_p*" -and $_.Name -notlike "*OLD*" -and $_.Name -notlike "*Test*"
        }
    }
    
    # Get existing shares
    $compartilhamentos = Get-WmiObject -ComputerName $nomeServidor -Class Win32_Share | Where-Object {
        $_.Path -like "D:\*" # ... more filters
    }
    
    # For each folder found
    foreach ($pasta in $listaPastas) {
        $pastaCompartilhada = $false
        
        # Check if it's already shared
        foreach ($compartilhamento in $compartilhamentos) {
            if ($pasta.FullName -eq $compartilhamento.Path) {
                $pastaCompartilhada = $true
                break
            }
        }
        
        # If not shared, create the share
        if (-not $pastaCompartilhada) {
            # Get current permissions
            $acl = Invoke-Command -ComputerName $nomeServidor -ScriptBlock {
                param ($pasta)
                $acl = Get-Acl -Path $pasta.FullName
                # Filter specific groups (_HCM_, _RH_, _COMPLIANCE_, _SAPIENS_)
                $filteredGroups = $acl.Access | Where-Object { 
                    $_.IdentityReference -match "_HCM_|_RH_|_COMPLIANCE_|_SAPIENS_"
                } | Select-Object -ExpandProperty IdentityReference
                return $filteredGroups
            } -ArgumentList $pasta
            
            # Classify groups by type
            $grupoRH = @()
            $grupoHCM = @()
            $grupoCompliance = @()
            $grupoSapiens = @()
            
            foreach ($group in $acl) {
                if ($group -match "_RH_") { $grupoRH += $group -replace '.*\\(.*?)$', '$1' }
                elseif ($group -match "_HCM_") { $grupoHCM += $group -replace '.*\\(.*?)$', '$1' }
                elseif ($group -match "_COMPLIANCE_") { $grupoCompliance += $group -replace '.*\\(.*?)$', '$1' }
                elseif ($group -match "_SAPIENS_") { $grupoSapiens += $group -replace '.*\\(.*?)$', '$1' }
            }
            
            # Define share permissions
            $gruposFull = @("MEGACLOUD\CloudOps", "MEGACLOUD\HCM MANAGER")
            $grupoChangeAccess = @("MEGACLOUD\Resolvedores Cloud") + $grupoRH + $grupoHCM + $grupoCompliance + $grupoSapiens
            
            # Create the share
            $nomeCompartilhamento = $pasta.Name + "$"
            Invoke-Command -ComputerName $nomeServidor -ScriptBlock {
                param ($nomeCompartilhamento, $caminhoPasta, $gruposFull, $grupoChangeAccess)
                New-SmbShare -Name $nomeCompartilhamento -Path $caminhoPasta -FullAccess $gruposFull -ChangeAccess $grupoChangeAccess | Out-Null
            } -ArgumentList $nomeCompartilhamento, $pasta.FullName, $gruposFull, $grupoChangeAccess
            
            Write-Host "The folder $($pasta.Name) was shared as \\$nomeServidor\$nomeCompartilhamento." -ForegroundColor Green
        }
    }
    
    # Restart option
    $restartScript = Read-Host -Prompt "Do you want to restart the script? Yes(Y) No(N)"
    if ($restartScript -ne "Y") { break }
}

Conclusion

This script is a practical example of applying PowerShell to administer and automate complex administrative tasks, improving efficiency and accuracy in the production environment.

Result: Automated management of network shares on production servers with preservation of existing NTFS permissions and proper configuration of access groups.
